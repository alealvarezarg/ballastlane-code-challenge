[$speckit-specify]

Create an initial project specification focused **only on the scaffolding and project structure** for a .NET Task Management System.

Do not define implementation details, business rules, API endpoints, frontend behavior, or testing scenarios yet. Those will be specified in future iterations.

The goal is to establish a clean and extensible foundation following Clean Architecture principles.

The specification should cover only:

* Solution structure
* Project organization
* Layer responsibilities
* Dependency direction
* Folder organization
* Basic naming conventions
* High-level architectural decisions required for the initial setup

Keep the specification **extremely short and concise**. Every section should be easy to review in a few seconds.

Avoid introductions, long explanations, unnecessary context, duplicated information, or "nice to have" text that does not provide direct value. Prefer bullet points over paragraphs whenever possible.

The objective is to produce a specification that contains only the essential information needed to start the project while remaining easy to maintain and evolve.

If necessary, update or simplify any generated files or templates to ensure they follow this philosophy of brevity and clarity.
