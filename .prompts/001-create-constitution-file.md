Create a simple `CONSTITUTION.md` for this project.

The constitution should define a few clear development principles that prioritize clean, maintainable, and testable code without adding unnecessary complexity.

Include principles for:

* Clean Architecture
* Separation of business logic
* Test-Driven Development
* Simplicity over complexity
* Dependency Injection
* Consistent API design
* Code readability
* AI-generated code must always be reviewed
* Basic project documentation

Each principle should have a short explanation of why it exists.

Keep the document concise, practical, and suitable for a small technical assessment rather than a large enterprise project.
